Android编码规范之命名

发表于 2017-09-04 | 阅读次数:

通用命名规范
特定类型命名规范

通用命名规范

所有的标识符只允许使用 ASCII 字符和数字。

特定类型命名规范

包命名

建议采用如下规则：【com】.【公司名/组织名】.【项目名称】.【模块名】

example：com.heren.mobilecare.login包名只允许使用小写字母和数字，并且单词直接连接（不允许使用下划线）

包分层规则

按照功能分包：

例如 com.heren.mobilecare.login,com.heren.mobilecare.register。

按照类型分包：

例如 com.heren.mobilecare.activity,com.heren.mobilecare.fragment.

类命名

Android 中类的命名与 Java 开发中采用相同的规范。采用帕斯卡命名法。类名通常使用 名词或名词短语，比如：Customer、ImmutableList。 单元测试类的命名规则必须以被测试类名开始，Test 单词结束。

Android 中常见类命名:

类型 样例
Application XxxApplication.java
Activity XxxActivity.java
Fragment XxxFragment.java
Service XxxService.java
BroadcaseReceiver XxxReceiver.java
ContentProvider XxxProvider.java
Adapter XxxAdapter.java
Handler XxxHandler.java
Interface IXxxInterface.java
Implement XxxImpl.java
Presenter XxxPresenter.java

方法命名

方法命名采用驼峰命名法，首字母小写，其他单词首字母均大写。方法名通常使用动词 或动词短语，比如：sendMessage,stop。 单元测试方法中可以含有下华夏，通常用来置顶特定场景：test<测试方法名>_场景， 比如 testPop_emptyStack

常量命名

常量命名所有字母均大写，单词之间用下划线分割

变量命名

变量命名必须采用驼峰命名方式，通常使用名词和名词短语。

非 public 非 static 的变量可以使用 m 开头

非常量的 static 变量可以使用 s 开头。

参数命名

参数命名采用驼峰命名。 严格限制使用单字符命名参数。某些约定俗成不会发生理解偏差的场景可以允许使用单 字符参数，例如使用 x,y 标识坐标的时候。
